# Guitar Pro 8 对齐计划

更新日期：2026-07-15

## 目标与验收口径

目标是把官方 Guitar Pro 8.1.2.37 导出的规则 PDF 恢复为可编辑、可播放、可回读校验的 GP5/IR。这里的“与 TuxGuitar 持平”只按同一批源曲、相同版式、相同语义评测器比较，不用目标曲回归成绩代替未知曲目的泛化成绩。

发布门槛分三层：

1. **配对渲染器门槛**：同源 source-disjoint 曲目上，事件 P/R 不低于 99%，核心事件、节奏和弦/品 exact 与 TuxGuitar 的差距均不超过 1 个百分点。
2. **Guitar Pro 难例门槛**：完整九曲独立 GP8 基准上，核心事件、节奏和弦/品 exact 均达到 98%，小节 exact 达到 95%。这比配对门槛更严格，当前尚未达到。
3. **导出门槛**：GP5 写出后必须回读；小节、声部事件、节奏、弦/品和可表示的延音必须 100% 保留。GP5 无法表达的语义要留在 IR，并在报告中逐项列出。

训练集、阈值选择集和验收曲按 `source_id` 隔离。`training_eligible=false` 的基准 PDF、标签、目标曲和任何人工分析结果不得回流训练。

## 为什么闭源不妨碍构造真值

Guitar Pro 8 只承担官方排版渲染，不需要从进程内读取私有 IR：

- GP3/GP4/GP5 的事件、时值、弦/品、延音和技法由 PyGuitarPro/TuxGuitar 读取；GP7/GP8 `.gp` 的 GPIF 语义由 alphaTab/GPIF 读取。
- 固定版本的官方 GP8 worker 批量导出 PDF 和页面布局。worker 只用于本机数据生成与验收，不进入 Git 仓库。
- 官方矢量 PDF 保留 TAB 数字、文字字体、字符边界框和绘图路径。它们提供实际渲染后的精确攻击坐标、弦位置、品位、速度、连音组数字和横梁结构。
- 源 GP 提供“是什么”，官方 PDF 提供“画在哪里”。二者对齐后即可得到训练监督，不需要 Guitar Pro 的内部私有对象。
- 扫描图或被栅格化的 PDF 没有矢量层，自动切换到检测器/CNN；不能把矢量 PDF 指标外推到这类输入。

## 六条工作流

### 1. 官方渲染与数据治理

对每首合法 GP 源文件固定生成 `score_tab`、`tab_only`、`score_only` 三种官方 PDF、页面 PNG、源语义 JSON 和版本/哈希清单。训练划分按源曲绑定，禁止同曲不同页面或不同版式跨集合泄漏。每次扩充先跑文件完整性、页数、小节数和语义覆盖审计，再允许进入训练。

当前已完成官方 GP8 批处理和 source-disjoint 基准。下一步是扩大技法覆盖采样，而不是无差别重复加入普通四分/八分音符。

### 2. 输入域路由与几何恢复

先判断 PDF 是否具有可信矢量层和 GP8 字体签名，再选择：

- GP8 矢量路径：字符/绘图结构为主，CNN 补缺；
- TuxGuitar 矢量或规则整页图片：现有几何 + CNN；
- 栅格 PDF/图片：纯检测器/CNN，并降低置信度声明。

页面继续按谱表、系统、小节、事件分层处理，不切成互相独立的“单符号袋”。版式模型必须分别验收 `score_tab`、`tab_only` 和 `score_only`。

### 3. 核心事件、节奏与弦品

矢量 TAB 文字优先确定攻击事件和弦/品；五线谱 CNN 定位休止、未打印延音和节奏。小节容量、拍号、连音组和重复片段作为序列约束。GP8 的黑色四边形横梁路径用于确认两层横梁的十六分音符，避免重复训练 CNN 去猜一个 PDF 已明确编码的结构。

本轮已加入：矢量攻击真值、重复伪事件清理、事件重编号后的 `tie_edges` 重映射、两层横梁时值覆盖、速度文字和小节精确闭合。

### 4. 关系型技法与参数

技法不是单事件布尔分类。下一阶段按关系/参数建模：

- tie、hammer/pull、slide：起点、终点、弦和目标品位；
- bend：弯音量（1/4、1/2、full 等）和曲线点，而非仅 `bend=true`；
- palm mute、let ring：起止区间；
- vibrato、harmonic、ghost、accent、staccato、pick stroke：事件或音符归属；
- repeat、volta、D.C./D.S./Coda：演奏顺序图。

关系候选由视觉/矢量结构提出，小型序列网络只做上下文消歧。未达到每类最小 support 和 precision 门槛的输出保持 unresolved，不静默猜测。

### 5. 三种 PDF 版式

优先级为：

1. `score_tab`：信息最完整，先达到官方 GP8 与 TuxGuitar 配对持平；
2. `tab_only`：复用矢量数字、横梁、小节容量和关系模块，训练独立的纯 TAB 休止/节奏模型，达到相同端到端门槛；
3. `score_only`：恢复音高、节奏和技法，但单凭五线谱无法唯一确定吉他弦/品。默认输出等音指法估计并显式标注不确定性；只有输入含指法/弦号等额外证据时才承诺原指法重建。

### 6. 回归、失败样本和发布

每轮自动执行：官方 PDF 生成 → PDF/图片 OCR → IR → GP5 → TuxGuitar 回读 → 官方 GP8 回渲染 → 源语义逐事件比较。失败按几何、额外/漏事件、节奏、弦品、延音、技法参数和导出损失分类。规则只有在独立 GP8 提升且 TuxGuitar 回归不下降时才保留。

LLM/OCR 大模型不作为主视觉识别器。若规则和轻量 CNN 的剩余错误集中在长程演奏顺序或关系消歧，再加入小型 Transformer/蒸馏序列模型；输入是结构化候选和置信度，不是整页图片自由生成 GP。

## 2026-07-15 当前结果

### 数据与轻量模型

- GP8 高可信节奏集：18,097 train / 2,596 validation / 793 test；其中 14,928 个 GP8 高可信事件、29,262 个矢量 TAB token、253 页和 2,749 个矢量锚定小节。4,227 个无可靠锚点的弱事件未加入训练。
- 节奏模型：GP8 validation 全语义 exact 99.307%，TuxGuitar source-disjoint test 全事件 exact 94.578%、可见主声部 97.462%。
- 延音模型：validation presence F1 100%，独立 test presence F1 93.3%。最终关系仍需矢量/音高/小节约束，不把该单项指标等同于端到端延音恢复率。

### 独立 GP8 九曲难例

`score_tab` 共 9 首、431 小节，均为 `training_eligible=false`：

| 指标 | 当前结果 |
| --- | ---: |
| 小节数 | 431 / 431 |
| 事件 precision / recall | 99.781% / 99.750% |
| 核心事件 exact | 90.658% |
| 节奏 exact | 97.618% |
| 弦/品事件 exact | 92.759% |

这套更难的九曲绝对指标还没有达到最终 98% 门槛。主要剩余误差是未打印延音音符、复杂和弦中的音高关系以及少量技法/结构参数。

### 同源三曲 GP8 与 TuxGuitar 配对

两边使用相同 3 首源曲、122 小节、1,196 个参考事件：

| 渲染器 | 事件 P/R | 核心事件 exact | 节奏 exact | 弦/品 exact | 小节 exact |
| --- | ---: | ---: | ---: | ---: | ---: |
| Guitar Pro 8 | 100% / 99.916% | 97.741% | 97.992% | 99.582% | 90.164% |
| TuxGuitar | 100% / 100% | 98.495% | 98.913% | 99.415% | 91.803% |

核心、节奏和弦/品已达到“与 TuxGuitar 差距不超过 1 个百分点”的阶段门槛；小节 exact 仍差 1.64 个百分点，继续作为下一轮节奏序列任务。

### 指定目标曲

`finaltest/test.gp5` 的官方 `GT.pdf` 回归：34/34 小节、278/278 事件、节奏 exact 100%、核心事件 exact 98.201%、弦/品事件 exact 98.201%，音符 precision/recall 为 100%/98.361%，30/34 小节完全一致。当前缺少 5 个第三弦延音音符；bend F1 88%、slide F1 100%，bend 量值、hammer 和 vibrato 仍未完整恢复。

产物位于 `lab/20260715_gp8_target_round22/`。该曲是明确分析过的回归曲，不作为独立泛化成绩。

## 下一阶段顺序

1. 把 `score_tab` 九曲难例的未打印延音恢复到弦/品 exact ≥98%，同时保持配对基准不回退。
2. 为 bend 量值、slide/hammer 端点和区间技法扩展 IR 与 GP5/GPIF 写出；分别统计 presence、relation、parameter 三层指标。
3. 把本轮矢量攻击/横梁/小节约束迁移到官方 GP8 `tab_only`，重建 source-disjoint 配对基准并达到同一门槛。
4. 增加重复、volta 和跳转结构；恢复标题、速度变化、调号与分页元数据。
5. 最后扩充栅格化官方 PDF 和不同 GP8 打印设置；其他软件/版本作为独立视觉域，不混入 GP8 指标。
